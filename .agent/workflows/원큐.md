description: (slash_one_queue / 원큐) Automated workflow for Q1 seasonal image generation (Batch Mode of generate-stock-image)
---
This workflow (`/원큐`) automates the Adobe Stock generation process in **batch mode**.

> [!CAUTION]
> **이미지 분석은 생략할 수 없습니다.** 모든 이미지는 반드시 `view_file`로 직접 확인하고, 분석 결과 기반으로 25~35개 키워드를 작성해야 합니다.

### Workflow Steps

#### 0. Environment Check
Ensure the `seasonal_prompts.json` is up to date with your planned prompts.

#### 1. Metadata & Folder Preparation
Creates the session timestamp and the initial metadata JSON files.
**Output:** A timestamp string (e.g., `2026-01-08_12-23-26`). Save this.
```bash
python process_seasonal_metadata.py
```

#### 2. Image Generation (Batch)
**Agent Action:**
Read `seasonal_prompts.json`. For each item:
1.  Run `generate_image` tool.
2.  **Prompt:** Use the `prompt` field (already configured with style/safety rules).
3.  **ImageName:** Use the `filename_base` derived from the ID/Subject (check terminal output of Step 1).

#### 3. Image Moving
Moves the generated images from the agent's artifact folder to the project generation folder.
Replace `<TIMESTAMP>` with the timestamp from Step 1.
```bash
python move_generated_images.py <TIMESTAMP>
```

#### 4. ⛔ 이미지 분석 & 키워드 보강 (병렬 — 필수!)

> [!CAUTION]
> **이 단계를 건너뛰면 키워드가 ~10개로 제한됩니다. 반드시 수행하세요.**

**Agent Action — 필수 승인 절차:**
0. **[MANDATORY]** 스크립트 임의 실행을 방지하기 위해, 작업을 시작하기 전 반드시 `notify_user` 도구를 통해 다음 메시지로 사용자의 승인을 받으세요:
"지금부터 스크립트 실행 없이, 이미지를 화면에 띄워 '직접' 수동 분석을 시작하겠습니다. 진행할까요?"

**Agent Action — 병렬 처리:**

1. `generations/<TIMESTAMP>/` 폴더 내 모든 이미지를 **병렬 `view_file` 호출**로 동시 분석
2. 각 이미지에 대해 아래 분석표 작성:

| 항목 | 분석 결과 |
|------|----------|
| 오브젝트 | _________ |
| 배경/장소 | _________ |
| 분위기 | _________ |
| 색상 | _________ |
| 조명 | _________ |
| 구도 | _________ |

3. 분석 결과 기반으로 **키워드 25~35개** 생성 (중요도 순 정렬)
   - Tier 1 (1~10): 핵심 오브젝트, 분위기, 시즌
   - Tier 2 (11~25): 장소, 색상, 조명, 스타일
   - Tier 3 (26~35): 용도(banner, background), 감정, 관련 개념
4. 제목도 이미지 기반으로 자연스러운 문장으로 재작성 (70자 이내)
5. 기존 JSON 파일의 `title`과 `keywords` 필드를 **덮어쓰기**

**병렬 처리 흐름:**
```
# 1차: 모든 이미지 병렬 view_file (동시 분석)
view_file(image_1.png) | view_file(image_2.png) | ... | view_file(image_12.png)

# 2차: 분석 결과 기반으로 모든 JSON 병렬 업데이트
write_to_file(image_1.json) | write_to_file(image_2.json) | ... | write_to_file(image_12.json)
```

#### 5. Processing Pipeline (Crop & Upscale)
Replace `<TIMESTAMP>` with the timestamp from Step 1.
```bash
python generation_pipeline.py <TIMESTAMP>
```

#### 6. CSV Generation
> [!IMPORTANT]
> **체크포인트**: JSON 내 keywords가 25개 미만인 파일이 있으면 **Step 4로 돌아가서 보강**.

Replace `<TIMESTAMP>` with the timestamp from Step 1.
```bash
python generate_submission_csv.py <TIMESTAMP>
```

#### 7. Final Review
Opens the upscaled folder for final visual verification.
Replace `<TIMESTAMP>` with the timestamp from Step 1.
```bash
explorer "generations\<TIMESTAMP>\upscaled"
```
